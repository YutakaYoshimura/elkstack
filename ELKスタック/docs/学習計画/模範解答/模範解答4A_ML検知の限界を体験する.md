# 模範解答 4A: 「緩やかな劣化」をMLは検知できるのか？

> **注意**: この模範解答は、問題4A の Step 4 まで自分で取り組んでから読むこと。

---

## Step 1 の解説: ML が前兆期間をどう認識しているか

### シングルメトリックビューアーで確認すべきこと

#### Typical（期待値）の変化

pump-01 の故障前兆期間（1/13〜1/19）において、**Typical は Actual の上昇に追従して徐々に上がっていく**。

これが「オンライン学習」の動作:
- 1/13: Actual が少し上がる → ML は「やや高いが許容範囲」と判断
- 1/14: さらに上がる → ML は「新しい水準に移行した」と学習し始める
- 1/15〜1/19: ML の Typical が Actual に追いつく → **乖離が小さいため異常スコアが低い**

```
日付    Actual（実際値の傾向）  Typical（ML の期待値の傾向）  乖離
1/10    3.0                    3.0                          ≈ 0（正常）
1/13    3.2                    3.0                          0.2（小さい）
1/15    3.5                    3.3 ← 追従開始               0.2（小さい）
1/18    4.2                    4.0 ← 追従済み               0.2（小さい）
1/20    1.5 ← 故障で急落       4.2                          2.7（大きい）→ 異常検知！
```

#### 異常スコア

- 1/13〜1/19: **スコアは 0〜10 程度**（低い or スコアなし）
  - ML は「緩やかな上昇」をトレンドとして吸収しているため
- 1/20: **スコア 90 以上**（高い）
  - 故障による急落は「突然の変化」なので、ML が検知できる

#### モデルバウンドの変化

- 1/13 以降、モデルバウンド（グレーの帯）は **上方向にシフト** する
- これは ML が「正常値の範囲が上に移った」と学習したことを示す

### 結論

**ML のオンライン学習は、7日間かけた緩やかな振動上昇を「新しいトレンド」として吸収してしまう。**
これが「茹でガエル現象」であり、ML が緩やかな劣化を検知できない根本原因。

## Step 2 の解説: 学習期間を制限しても結果は同じ

### なぜ期間制限が効かないのか

Elasticsearch ML のオンライン学習は **ジョブが実行されている間、常にモデルを更新し続ける**。

「1/1〜1/12 のデータだけで学習させる」という設定は存在しない。
ジョブを 1/1〜1/25 の範囲で実行すると、1/13 以降のデータもモデルに取り込まれる。

### 期間制限ジョブ（week5-normal-only）の結果

全期間ジョブとほぼ同じ結果になる:
- 1/13〜1/19: スコアは低い（前兆を検知できない）
- 1/20: 急落を検知（高スコア）

### この結果が意味すること

**Elasticsearch ML では「学習データの固定」ができない。**

教師あり学習（Python の scikit-learn など）では「学習データ」と「テストデータ」を分けることができるが、
Elasticsearch ML の教師なし学習はオンライン学習が前提のため、この分離ができない。

## Step 4 の模範解答: 結論

### ML vs 固定閾値

```
ML の強み:
  - 正常パターンを自動学習するため、設備ごとの手動設定が不要
  - 急激な変化（急落・スパイク）は高精度で検知できる
  - 周期パターン（時間帯変動）を考慮した検知ができる

ML の弱み:
  - 緩やかな劣化（7日間かけた上昇）をトレンドとして吸収してしまう
  - 「学習データの固定」ができないため、異常データもモデルに取り込まれる
  - 前兆検知（予知保全の核心）が苦手

予知保全での使い方の提案:
  ML と固定閾値を併用する。
  - 固定閾値: 緩やかな劣化の検知（avg+3σ など）
  - ML: 急激な変化、未知のパターン、複数センサーの相関異常の検知
  - 両方のアラートを統合して、総合的に判断する
```

### 「MLだけで予知保全」は実現可能か？

**現状の Elasticsearch ML だけでは不十分。**

緩やかな劣化パターン（予知保全で最も重要なケース）を検知するには、
固定閾値または Python ML（教師あり学習）との併用が必要。

ただし、ML が「完全に無力」というわけではない:
- 急激な故障や異常パターンの検知は ML の方が得意
- 設備が多数ある場合、全設備に手動で閾値を設定するのは現実的でないため、ML による自動学習が有効
- **ML と閾値は相互補完の関係** であり、どちらか一方ではなく両方を使うのが最適解

## 発展的な視点

### 「MLの弱点を補うML」は可能か？

Elasticsearch ML のオンライン学習では対応できないが、以下のアプローチなら対応可能:

1. **教師あり学習（Python ML）**: 「正常」「前兆」「故障」のラベル付きデータで学習
   → 緩やかな変化パターンも学習できる（ただし故障データが必要）

2. **変化点検知**: 値そのものではなく「変化の傾きが変わった点」を検出する手法
   → 1/13 時点で傾きが変わったことを検知できる可能性

3. **移動平均の比較**: 短期移動平均と長期移動平均の乖離を監視
   → 短期が長期を上回り続けたらアラート

これらは Week 5 の範囲を超えるが、予知保全の精度を追求する場合に検討する価値がある。
