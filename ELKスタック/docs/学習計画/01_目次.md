# ELKスタック 予知保全 1ヶ月学習カリキュラム 目次

> **対象**: ELKスタック（Elasticsearch + Logstash + Kibana）初心者
> **前提**: Step 1〜6 を一通り実施し「動くのは見えた」状態
> **ゴール**: 1ヶ月後に「自分で考えて、自分で作れる」状態になる

---

## カリキュラム全体構成

```
Week 1（Day  1〜 7）: データの流れを目で追い、正常値の特性を把握する
Week 2（Day  8〜14）: 単一センサーで「異常を検知する」（ML アプローチ）
Week 3（Day 15〜21）: 「閾値をどう決めるか」を考え、「異常をメールで通知する」★核心
Week 4（Day 22〜30）: 複数センサーへ展開し、全体を統合する
```

## 学習の進め方

### 推奨する読み方

```
1. まず「00_提案.md」で全体方針を理解する
2. 各週のファイルを順番に読み、手を動かしながら進める
3. Week 3 の Day 17 になったら「問題1」に挑戦する
4. 必ず自分で考えてから模範解答を読む
```

### 1週間の目安ペース

| 日 | 目安の学習時間 | やること |
|---|---|---|
| Day 1 | 1〜2時間 | 週別ファイルを読み、最初の課題に着手 |
| Day 2〜6 | 1〜2時間/日 | 課題を実施し、アウトプット課題をこなす |
| Day 7 | 2時間 | 週の仕上げ課題、振り返り |

### 困ったときの参照先

| 困りごと | 参照先 |
|---|---|
| ELK の基本概念がわからない | `docs/02_より理解をすすめるために.md` |
| 次の進め方で迷った | `docs/03_今後の進め方.md` |
| コマンドを忘れた | プロジェクトルートの `CLAUDE.md` |
| Logstash が動かない | `docs/01_学習手順.md` のトラブルシューティング |
| ElastAlert2 のルール文法 | 各週別ファイルのコラム、または公式ドキュメント |

## 1ヶ月後の到達目標チェックリスト

このカリキュラムを終えたとき、以下をすべて「自分でできる / 説明できる」状態を目指す。

```
データの流れ
  □ CSV → Logstash → Elasticsearch の流れを図で説明できる
  □ Logstash の input/filter/output の役割を説明できる
  □ インデックス・ドキュメント・マッピングを自分の言葉で説明できる
  □ Elasticsearch API（_search, extended_stats）を自分で書ける

単一センサー可視化
  □ Lens で時系列グラフをゼロから作れる
  □ 集計方法（平均・最大・最小）を状況に応じて使い分けられる
  □ KQL フィルターで設備・期間を絞り込める
  □ ダッシュボードに複数パネルを組み合わせられる

異常検知（ML）
  □ Elasticsearch ML の Single / Multi-metric ジョブを自分で作れる
  □ 教師なし学習がなぜ異常を検知できるかを説明できる
  □ Anomaly Explorer の Actual / Typical / Anomaly score の読み方を説明できる
  □ 振動だけで検知できる故障とできない故障を説明できる

閾値設計
  □ 正常期間の統計（avg・max・std）を API で自分で調べられる
  □ 「瞬間値（type: any）」と「frequency（複数件）」のどちらを使うべきか根拠を説明できる
  □ 「固定閾値」「avg+3σ」「変化量ベース」の特徴と使い分けを説明できる
  □ 自分が設定した閾値と模範解答の違いとその理由を説明できる
  □ 「誤検知と検知漏れのトレードオフ」を説明できる
  □ センサーの種類によって閾値設計のアプローチが変わることを説明できる
  □ 「ロジックをコピー」することと「数値をコピー」することの違いを説明できる

アラート通知
  □ ElastAlert2 のルールファイルをゼロから書ける
  □ 閾値ベースのアラートと ML ベースのアラートの違いを説明できる
  □ 実際にメール通知を設定して届けることができた
  □ 「アラート疲れ」とその対策（realert, frequency）を説明できる

複数センサー
  □ センサー間の相関（電流↑→温度↑など）を実データで確認できた
  □ 故障モードによって「最初に反応するセンサー」が違うことを確認できた
  □ Multi-metric ジョブを作り、単一センサーと比較できた
  □ 3台の設備を一覧できるダッシュボードを作れた
```
