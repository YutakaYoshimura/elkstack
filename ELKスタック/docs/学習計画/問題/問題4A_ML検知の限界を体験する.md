# ML実践問題 4A: 「緩やかな劣化」をMLは検知できるのか？

> **タイミング**: Week 5 の Day 34〜36 で取り組む
> **前提**: Week 2 の sensor-anomaly-detection ジョブが存在すること

---

## この問題で学ぶこと

- ML の「オンライン学習」が緩やかな変化にどう反応するかを **自分の目で確認する**
- 「予知保全にMLだけで十分か？」という問いに、データで答えを出す
- シングルメトリックビューアーの「モデルバウンド」を読み解く力を鍛える

---

## 問題の背景

Week 2 で、ML は pump-01 の故障1（1/20 ベアリング劣化）について
**故障当日の急落は検知したが、7日前からの緩やかな振動上昇は検知できなかった** と学んだ。

「これでは予知保全にならないのでは？」という疑問が残っている。

この問題では、その疑問を **自分で検証し、結論を出す**。

---

## Step 1: ML が故障前兆をどう認識しているかを詳細に確認する

### 1-1. シングルメトリックビューアーで故障前兆期間を拡大表示する

1. **機械学習** → **異常検知** → **ジョブ管理**
2. `sensor-anomaly-detection` の **シングルメトリックビューアー** を開く
3. 時間範囲を **2026-01-10 〜 2026-01-22** に設定（前兆期間を中心に）
4. Detector: `mean(vibration)`、設備: `pump-01` を選択

### 1-2. 以下の情報を読み取ってメモする

```
□ 1/10〜1/12（正常期間）の Typical（期待値）はいくつか？
□ 1/13（前兆開始）以降、Typical はどう変化しているか？
  → 上昇している？ 横ばい？
□ 1/13〜1/19 の Actual（実際値）は Typical と比べてどうか？
  → 大きく乖離している？ それとも Typical に近い？
□ 1/13〜1/19 で異常スコアが付いているバケットはあるか？
  → ある場合、スコアはいくつか？
□ モデルバウンド（グレーの帯）は 1/13 以降どう変化しているか？
  → 帯が広がっている？ 狭いまま？
```

### 1-3. ここで考えてほしいこと

```
問い: ML の Typical（期待値）が Actual（実際値）の上昇に追従しているなら、
      それは何を意味するか？

ヒント: オンライン学習は「最新のデータでモデルを常時更新する」仕組み
```

**この時点で、自分なりの結論を書き出してから Step 2 に進む。**

## Step 2: 「正常期間だけで学習した場合」と比較する

Step 1 で見たジョブは、**故障期間も含めた全データで学習** している。
では「正常期間のデータだけで学習させたら、前兆を検知できるのか？」を確かめる。

### 2-1. 新しいジョブを作成する（学習期間を制限）

1. **機械学習** → **異常検知** → **ジョブを作成**
2. データビュー `sensor-data` を選択
3. **シングルメトリック** を選択
4. 時間範囲を **2026-01-01 〜 2026-01-25** に設定（カスタム範囲）
5. Field: `Mean(vibration)`
6. Split field（by フィールド）: `device_id.keyword`
7. バケットスパン: `15m`
8. ジョブID: `week5-normal-only`
9. **ジョブを作成**

### 2-2. 結果を確認する

1. シングルメトリックビューアーを開く
2. 時間範囲: **2026-01-10 〜 2026-01-22**
3. pump-01 の vibration を表示

### 2-3. 以下を確認してメモする

```
□ 1/13〜1/19 で異常スコアはついているか？
□ Week 2 のジョブ（全期間学習）と比較して、検知結果に違いはあるか？
□ モデルバウンドの広さに違いはあるか？
```

> **重要な気づき**: ジョブの学習期間を制限しても、Elasticsearch ML は
> **ジョブ開始時点からオンライン学習を始める** ため、前兆期間のデータも
> 学習に取り込まれる。学習データを「正常期間のみ」に完全固定する仕組みは
> Elasticsearch ML には存在しない。

## 補足: 「MLだけでは不十分」は欠陥ではない

Elasticsearch ML のオンライン学習は「環境変化に適応する」ために設計されている。
工場の稼働条件が変わったとき（季節変化、生産量変更など）に、自動で追従できるのが強み。

しかしその結果、**緩やかな劣化も「新しい正常」として取り込んでしまう** という弱点がある。

これは Elasticsearch ML の「欠陥」ではなく「設計上のトレードオフ」。
予知保全の現場では **ML と固定閾値を併用する** のが一般的な解決策。

```
ML の役割:      急激な変化、未知のパターンの検出
固定閾値の役割: 緩やかな劣化、絶対的な上限の監視
```
