# 閾値設計 問題3: motor-01 振動センサー × 横展開

---

## この問題で学ぶこと

- 「同じ手法を別の設備に展開する（横展開）」ときの正しいやり方を理解する
- 「ロジック（考え方）のコピー」と「数値のコピー」を混同しないようにする
- 設備ごとに統計を取り直すことの重要性を体験する

---

## 問題の背景

問題1で pump-01 の振動センサーに対する閾値設計を学んだ。
今度は**別の設備 motor-01** でも同じ故障モード（ベアリング劣化）が発生している。

> **この問題の核心:**
> 「同じ故障モード（ベアリング劣化）だから、pump-01 で決めた閾値の数値を
> motor-01 にそのままコピーすれば良い」と思うのは**危険**。
>
> なぜ危険なのか、を自分で確かめるのがこの問題の目的。

**使うデータ**:
- 設備: `motor-01`
- センサー: `vibration`
- 故障日: 2026-02-05
- 正常期間（閾値の基準にする）: 2026-01-01〜2026-01-25

---

> ### コラム: 「同じ数値を使い回すことの危険性」
>
> 工場の予知保全システムを構築するとき、こんな場面がよくある：
>
> 「pump-01 の閾値は 4.5 に設定したら上手くいった。
>  pump-02 も同じポンプだし、4.5 でいいんじゃない？」
>
> **これは正しいケースもあるが、間違いのケースも多い。**
>
> 同じメーカー・同じ型番の機械でも、実際に動作する環境によって振動値の水準が変わる：
> - 設置場所の振動（隣の機械の振動が伝わるなど）
> - 経年劣化の度合い（古い機械 vs 新しい機械）
> - 稼働条件（回転数・負荷量の設定値）
> - ベースラインのばらつき（機械ごとの個体差）
>
> 正しい横展開の考え方：
>
> | 横展開すべきもの | 横展開してはいけないもの |
> |---|---|
> | 「avg + 3σ を使う」というロジック | avg + 3σ の計算結果（具体的な数値） |
> | 「正常期間でキャリブレーションする」という方針 | 他の設備で算出した閾値の数値 |
> | ElastAlert2 のルール構造・書き方 | `gte` に入れる具体的な値 |
>
> この問題では、**「数値をコピーしたらどうなるか」を先に体験してから**、
> 「正しい横展開」に取り組む。

## Step 0: pump-01 の閾値を motor-01 にそのまま適用してみる（先にやる）

問題1で自分が設定した閾値の値を、motor-01 のルールにそのまま流用する。
`device_id.keyword` を `motor-01` に変えるだけ。

ファイルのパス: `elastalert/rules/motor01-pumpcopy.yaml`

```yaml
name: "motor-01 振動（pump-01の閾値をそのまま流用）"
type: any
index: sensor-data-*

filter:
  - term:
      device_id.keyword: "motor-01"    # ← pump-01 から変えるだけ
  - range:
      vibration:
        gte: 5.0    # ← 問題1で決めた値をそのまま使う（自分の値に変更）

realert:
  minutes: 60

alert:
  - debug
```

### テスト実行（pump-01 の閾値を流用した場合）

```powershell
docker exec elastalert elastalert-test-rule `
  --start "2026-01-22T00:00:00" `
  --end "2026-02-06T00:00:00" `
  /opt/elastalert/rules/motor01-pumpcopy.yaml
```

## Step 2: motor-01 専用の閾値を設計する

問題1で学んだ「設計ロジック（avg + 3σ を使う）」を**そのまま採用し**、
ただし「数値」は motor-01 の統計から**新たに計算**する。

### 横展開の正しいやり方

```
【横展開のルール】

コピーするもの（正しい）:
  「avg + Nσ を閾値にする」というアプローチ（ロジック）
  「正常期間のデータをもとに計算する」という方針
  YAML のルール構造

コピーしてはいけないもの（危険）:
  「gte: 4.5」のような具体的な数値
  pump-01 の統計値（avg、std）
  pump-01 で算出した avg + 3σ の結果
```

### 閾値設計シート（motor-01 専用）

```
【motor-01 専用 閾値設計シート】

  （問題1で最も効果的だったσ倍率と同じにする）

  motor-01 の avg: ________（Step 1 で確認）
  motor-01 の std: ________（Step 1 で確認）
  motor-01 の avg + ___σ = ________ + ___ × ________ = ________（← これが閾値）

  pump-01 の閾値との比較:
    pump-01 の閾値: ________
    motor-01 の閾値: ________
    差: ________（差がある場合、それは正常・想定内）
```

### YAML ファイルを作成する

ファイルのパス: `elastalert/rules/motor01-vibration-my.yaml`

```yaml
name: "motor-01 振動閾値アラート（motor-01専用設計）"

index: sensor-data-*

  - term:
      device_id.keyword: "motor-01"
  - range:
      vibration:
        gte: 5.0    # ← motor-01 の統計から計算した値に変更する

  minutes: 60

  - debug
```

```powershell
  --start "2026-01-22T00:00:00" `
```
