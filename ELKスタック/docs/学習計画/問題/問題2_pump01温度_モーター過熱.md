## 問題の背景

pump-01 は 2026-02-15 にモーター過熱で停止した。
今度は温度センサー（`temperature` フィールド）を使って、この故障を事前に検知する閾値を設計する。

> **重要な前提**: 問題1（振動センサー）で学んだ手法（avg + 3σ）を温度センサーにそのまま適用するのが正しいとは限らない。なぜか、というのがこの問題のテーマ。

**使うデータ**:
- 設備: `pump-01`
- センサー: `temperature`
- 故障日: 2026-02-15
- 正常期間（閾値の基準にする）: 2026-01-22〜2026-02-08（故障1収束後から故障2の1週間前まで）

## Step 0: まずデータを「観察」する（ここは先にやる）

閾値を決める前に、温度センサーの**特性**を自分の目で観察する。
これは問題1でやらなかったステップ。

**なぜ問題2から「観察」のステップが必要になるのか？**

振動センサーは「機械的な摩耗・劣化」だけを反映するシンプルなセンサーだった。
しかし温度センサーは、機械の状態以外にも**様々な外部要因**の影響を受ける可能性がある。
それを先に確認しておかないと、誤った閾値を設計してしまうリスクがある。

### 観察1: 温度と電流の関係を確認する

Kibana の Discover で以下を確認する。

```
KQL クエリ: device_id.keyword : "pump-01"
時間範囲: 2026-01-22 〜 2026-02-08（正常期間）
```

Lens（可視化）を使って散布図を作成：
- 横軸: `current`（電流）
- 縦軸: `temperature`（温度）
- 1点 = 1つの計測値

**確認したいこと**: 電流が高いとき、温度も高くなる傾向があるか？

### 観察2: 時間帯による温度の変化を確認する

Lens で折れ線グラフを作成：
- 横軸: `@timestamp`（時刻）
- 縦軸: `temperature` の平均
- 集計単位: 1時間

**確認したいこと**: 昼間と夜間で温度の水準に差があるか？

> ### コラム: センサーの特性によって閾値設計のアプローチが変わる
>
> センサーには様々な種類があり、それぞれ異なる「特性」を持っている。
>
> **振動センサー（問題1）の特性:**
> - 主に機械部品の摩耗・損傷を反映する
> - 外部環境（気温、時間帯など）の影響を比較的受けにくい
> - 正常時の値は安定していることが多い
>
> **温度センサー（問題2）の特性:**
> - 機械の発熱だけでなく、外部気温・稼働時間・負荷量など多くの要因の影響を受ける
> - 稼働直後（暖機中）と安定稼働時で値が異なることがある
> - 昼間（高負荷）と夜間（低負荷）で水準が違う場合がある
>
> **電流センサー（参考）の特性:**
> - 負荷量（処理量）を強く反映する
> - 季節や時間帯よりも、何を処理しているかによって変動する
>
> この特性の違いを「センサー特性」と呼ぶ。
> 「センサー特性を無視して全てのセンサーに同じ手法を適用する」のは
> 実際の現場ではよくある失敗パターン。
>
> **この問題では、温度センサーの特性を自分で発見し、
> それを踏まえた閾値設計ができるかどうかを問う。**

## Step 1: 正常期間の統計を自分で調べる

```powershell

  "http://localhost:9200/sensor-data-*/_search?pretty" `
```

## Step 3: 実装してテストする

選んだ案を ElastAlert2 のルールファイルとして実装する。
YAML の書き方は問題1の「コラム: ElastAlert2 の YAML ルールファイルとは」を参照。

ファイルのパス: `elastalert/rules/temperature-threshold-my.yaml`

```yaml
name: "pump-01 温度閾値アラート（自分の設計）"
type: any
index: sensor-data-*

filter:
  - term:
      device_id.keyword: "pump-01"
  - range:
      temperature:
        gte: 65.0    # ← ここを自分が決めた値に変更する

realert:
  minutes: 120    # 温度は変化が遅いため長めに設定（変更可）

alert:
  - debug
```

### テスト実行

```powershell
docker exec elastalert elastalert-test-rule `
  --start "2026-02-01T00:00:00" `
  --end "2026-02-16T00:00:00" `
  /opt/elastalert/rules/temperature-threshold-my.yaml
```

テスト結果の読み方は問題1の「テスト実行結果の読み方」を参照。
