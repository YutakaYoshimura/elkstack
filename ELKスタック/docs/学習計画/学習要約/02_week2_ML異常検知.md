## Day 12〜14: Anomaly Explorer を深く読み込む

### 3つの故障の詳細分析

#### 故障1: pump-01, 2026-01-20 ベアリング劣化

| 項目 | 値 |
|---|---|
| Actual（実際値） | 3.05 |
| Typical（期待値） | 4.069 |
| Anomaly Score | 97 |
| 検知内容 | 1.3x 低い（故障後の急落を検知） |

**重要な発見**: MLが検知したのは故障前の上昇ではなく、**故障当日の急な低下**。
1/13〜1/19 の緩やかな振動上昇は、MLが「新しいトレンド」として学習してしまい、異常と判定されなかった。

シングルメトリックビューアーで確認:
- 1/17〜1/18: 「1.2x 高い」と認識はしていた（スコア < 1）
- 異常とまでは判断しなかった → **固定閾値（3.92）の方が約4日早く検知可能**

#### 故障2: pump-01, 2026-02-15 モーター過熱

- **振動センサーでは検知できず**
- 理由: 温度起因の故障は振動値に変化が出ない
- → 温度センサーの監視が必要

## Elasticsearch ML の仕組み

### 確率的モデル（回帰モデルではない）

- 各バケットで「この値が出る確率」を計算
- 確率が低い → 異常スコアが高い

### 学習する特徴

- トレンド（上昇・下降の傾向）
- 周期性（時間帯による変動パターン）
- 直近の値の推移

### オンライン学習

- 常に最新データでモデルを更新し続ける
- **強み**: 環境変化に適応できる
- **弱み**: 緩やかな異常変化を「新しい正常」として取り込んでしまう（茹でガエル現象）

---

## Actual / Typical / Anomaly Score の関係

```
Actual:  その時間帯に実際に計測された値
Typical: MLが「このくらいの値になるはず」と予測した値
Score:   Actual と Typical の乖離度合い（0〜100）

重要: スコアは絶対値ではなく「期待値からの乖離量」で決まる
→ 「値が高い」だけでは異常とは限らない
→ 「その時間帯にしては異常に高い（または低い）」かどうかが判断基準
```

## 異常検知結果レポート（vibration-only-detection）

### 故障1（pump-01, 1/20 ベアリング劣化）
- 事前検知: できなかった（故障当日の急落を検知）
- 固定閾値との比較: 閾値の方が約4日早く検知可能

### 故障2（pump-01, 2/15 モーター過熱）
- 振動センサーでの事前検知: できなかった
- 理由: 温度起因の故障は振動値に変化が出ないため

### 故障3（motor-01, 2/5 ベアリング劣化）
- 事前検知: できなかった（故障翌日の急落を検知）

### 総合評価

**MLの強み**: トレンドを含めた変化で異常を検知できる。設備ごとの正常パターンを自動で学習できる（設備が多数あっても手動設定不要）。

**MLの弱み**: 故障に向けて緩やかに変化している場合に予兆として捉えられない（オンライン学習が裏目に出る）。

**閾値との使い分け**: MLと閾値の長所・短所を理解した上でうまく併用していくことで予知保全の精度をあげることができる。
- ML が向いている場面: 正常パターンが設備ごとに異なり、手動で閾値設定が困難な場合。急激な変化の検知。
- 閾値アラートが向いている場面: 緩やかな劣化傾向の検知。明確な基準値がある場合。

## 運用での使い分け

### ダッシュボード vs ML分析画面

```
ダッシュボード（常時監視） → 異常に気づく → ML画面で詳細分析 → 対応判断
```

- **ダッシュボード**: センサー値の時系列 + 閾値ライン + 異常スコア推移 → 日常監視用
- **シングルメトリックビューアー**: モデルバウンド付きの詳細時系列 → 深掘り分析用
- **異常エクスプローラー**: 複数設備の異常スコア一覧 → 全体俯瞰用

シングルメトリックビューアーはダッシュボードに直接配置できない（ML専用画面）。
同等の情報は Lens の折れ線グラフ + 基準線で再現可能。

## Week 2 で学んだことの全体像

```
1. ML異常検知は「確率的モデル」で、正常パターンからの乖離を検知する
2. オンライン学習のため、緩やかな変化には追従してしまう弱点がある
3. 単一センサー（振動のみ）では、温度起因の故障は検知できない
4. MLと固定閾値は相互補完の関係 → 併用が最も効果的
5. Actual / Typical / Score の3つの値で異常の内容を読み解ける
```
