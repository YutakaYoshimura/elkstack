# センサー異常検知・予兆検知 調査まとめ

> 対象センサー：振動 / 温度 / 電流 / 圧力 / 流量
> 稼働パターン：24時間・一定負荷・連続稼働
> 基盤：ELKスタック（Elasticsearch / Kibana）

---

## 1. 診断の基本構成

### 一次診断と二次診断の役割

| 診断 | 目的 | 方針 |
|------|------|------|
| 一次診断 | 異変の疑いがあるセンサーデータを検知 | 見逃しを減らすことを優先。誤検知はある程度許容し、二次診断に渡す |
| 二次診断 | 一次で検知したものが本当に異常・予兆かを詳細分析 | 複数センサーの相関・過去パターンとの比較・重篤度分類を行う |

### 診断フロー

```
センサー値（振動・温度・電流・圧力・流量）
        ↓
【一次診断】異変の疑いを広く検知
  固定閾値 / 変化率 / 移動平均 / 単一ML
        ↓ 疑いありと判定したデータのみ
【二次診断】本当に異常・予兆かを詳細分析
  複合条件判定 / 多変量ML / 異常スコア分類
        ↓
判定結果：正常 / 要注意 / 要点検 / 即対応
```

## 2. 固定閾値 vs 機械学習（ML）比較

| 観点 | 固定閾値 | 機械学習（ML） |
|------|----------|----------------|
| 得意な検知 | 絶対値超過・物理限界・急激変化 | パターン変化・相関異常・未知の異常 |
| 季節・時間変動 | 苦手（別ルールが必要） | 自動で適応 |
| 緩やかなトレンド | 移動平均＋閾値で対応可 | **苦手**（学習し続けてしまう） |
| 説明しやすさ | ○ ルールが明確 | △ ブラックボックス |
| 導入の手軽さ | ○ すぐ使える | △ 学習データと期間が必要 |
| ELKライセンス | Basic（無料） | Basic（APM連携はPlatinum以上） |

### MLが苦手な緩やかなトレンド検知の補完手法

MLは「3ヶ月かけてじわじわ上昇」のような緩やかな変化を正常として学習し続けるため検知できない。以下の固定閾値手法で補完する。

| 手法 | 内容 |
|------|------|
| 移動平均＋固定閾値 | 過去30日の平均値がXを超えたらアラート |
| 変化率監視 | 1週間前比でY%以上変化したらアラート |
| ML Forecast（補助） | 周期パターンがある場合は「X日後に閾値を超える」という予測アラート |

## 3. 診断パターン一覧

### 固定閾値パターン

#### 一次診断

| ID | 名称 | 内容 | 向いているセンサー | ELKでの実装 |
|----|------|------|-------------------|-------------|
| A | 絶対値 上下限チェック | センサー値が上限・下限を超えたらフラグ。最もシンプルな一次検知 | 圧力（耐圧限界）、電流（過電流・断線）、温度（安全上限） | Kibana Alerting |
| B | 時間帯・季節別 閾値チェック | 昼夜・夏冬で異なる閾値を設定し、時刻・月に応じて切り替えて判定 | 温度（外気温連動） | Kibana Alerting ＋ 閾値テーブルIndex / Enrich Processor |
| C | 変化率・変化速度 チェック | 「直近N分でX以上変化」「前時間帯比でY%以上変化」を検知。急激な変化を捉える | 温度（急激な上昇）、圧力（急激な変動）、振動（急激な増加） | Kibana Alerting ＋ Painless Scriptで変化率計算 |
| D | 移動平均＋閾値（トレンド検知） | 過去N日の移動平均が閾値を超えたらアラート。MLが苦手な緩やかな劣化予兆の検知に有効 | 温度（じわじわ上昇）、電流（じわじわ増加）、振動（全体的な増加傾向） | Kibana TSVB / Lens の moving_average ＋ Alerting |

#### 二次診断

| ID | 名称 | 内容 | 活用例 | ELKでの実装 |
|----|------|------|--------|-------------|
| E | 複数条件の組み合わせ判定 | 複数センサーのルールを組み合わせて異常を絞り込む | 温度上昇＋電流増加 → 過負荷異常、振動増加＋温度上昇 → 軸受け劣化、流量低下＋圧力上昇 → 詰まり | Kibana Alerting（AND/OR条件）/ Watcher |
| F | 継続時間チェック | 「閾値超過がN分以上継続」している場合のみ異常と判定。瞬間スパイク（ノイズ）と区別する | 温度が閾値超過で5分以上継続 → 真の過熱異常 | Kibana Alerting（アクティブ期間設定）/ Watcher |

### 機械学習（ML）パターン

#### 一次診断

| ID | 名称 | 内容 | 向いているセンサー | 注意点 |
|----|------|------|-------------------|--------|
| G | 単一メトリクス Anomaly Detection | 1センサー値に対してMLがベースラインを学習し、外れたものを異常スコアで検出。季節性・周期性を自動で考慮 | 振動（負荷依存の変動）、流量（生産スケジュール連動） | 緩やかなトレンドは正常として学習し続けるため検知できない |
| H | Population分析（機器間比較） | 同種機器が複数台ある場合、他機器と比較して「この機器だけ値が高い・低い」を検知 | 全センサー（同種機器が複数台ある場合） | 同種の機器が複数台必要 |

#### 二次診断

| ID | 名称 | 内容 | 活用例 | ELKでの実装 |
|----|------|------|--------|-------------|
| I | 多変量解析（複数センサー相関） | 複数センサーを同時に学習し、個々の値は正常でも「組み合わせが普段と違う」パターンを異常として検出 | 温度・電流・振動がそれぞれ正常範囲内でも3つが同時に少し高め → 予兆の可能性 | Elastic ML → Multivariate Anomaly Detection Job |
| J | 異常スコアによる重篤度分類 | MLのAnomaly Score（0〜100）で自動分類する | 25〜50：要注意 / 50〜75：要点検 / 75以上：即対応 | Elastic ML ＋ Kibana Alerting（スコア閾値でアクション分岐） |
| K | Forecast（将来値予測） | 学習済みモデルから将来値を予測し「X日後に閾値を超える」という予測アラートを出す | 周期パターンの延長予測（昼夜・週次の周期がある設備） | Elastic ML → Forecast API / Kibana ML Forecast機能 |

> **重要：** 二次診断のMLは一次フラグが立ったデータだけでなく、全センサーデータで学習させること。一次通過データのみで学習すると正常パターンの学習データが偏り精度が落ちる。

## 4. センサー別 推奨パターン

### 振動

- **特性：** 機械の回転数・負荷に依存して変動。周波数パターンの変化が軸受け・ギアの劣化予兆になる。絶対値よりも「変化」を見るのが基本。
- **一次診断：** C（変化率）、G（単一ML）
- **二次診断：** I（多変量ML）、J（スコア分類）
- **備考：** 振動の絶対値閾値はほぼ意味がない。「いつもと違う振れ方」を捉えるMLが最も有効なセンサー。温度・電流との多変量解析で故障種別の絞り込みができる。

### 温度

- **特性：** 外気温に連動するため季節・時間帯の影響が大きい。安全上限（仕様値）は絶対値管理が必要。緩やかな上昇は劣化予兆になる。
- **一次診断：** A（絶対値・安全上限は必須）、B（季節別閾値）、D（移動平均）
- **二次診断：** E（複合条件）、I（多変量ML）
- **備考：** 安全上限チェック（A）は必須。それとは別に劣化予兆用のトレンド監視（D）を設けるのが理想。

### 電流

- **特性：** 同一負荷・同一動作であれば値が安定。定格電流から上下限が計算できる。季節の影響は小さい。
- **一次診断：** A（絶対値・過電流/断線）、D（移動平均・モーター劣化予兆）
- **二次診断：** E（複合条件）、I（多変量ML）
- **備考：** 電流のじわじわ増加はモーター劣化・軸受け異常の典型的な予兆。移動平均＋閾値（D）が有効。

### 圧力

- **特性：** 動作仕様で値が決まるため、季節・時間帯の影響をほぼ受けない。固定閾値との相性が最も良いセンサー。
- **一次診断：** A（絶対値・耐圧限界は必須）、C（変化率・急激な変動）
- **二次診断：** E（複合条件・流量との組み合わせ）、F（継続時間）
- **備考：** 流量との組み合わせ（E）で詰まり・リーク・ポンプ異常を切り分けられる。
  - 流量低下 ＋ 圧力上昇 → 詰まり
  - 流量低下 ＋ 圧力低下 → リーク・ポンプ故障

### 流量

- **特性：** 生産スケジュール・季節需要で正常値の幅が変わる。一定負荷型でも生産量変動がある場合は固定閾値が効きにくい。
- **一次診断：** C（変化率・急激な変化）、G（単一ML・変動パターン学習）
- **二次診断：** E（複合条件・圧力との組み合わせ）、I（多変量ML）
- **備考：** 生産量が本当に一定であれば固定閾値（A）も使えるが、少しでも変動があるなら変化率（C）かML（G）の方が安定する。

## 5. 推奨構成（24時間・一定負荷・連続稼働）

### 一次診断

| センサー | 推奨パターン |
|---------|-------------|
| 圧力 | A（絶対値）＋ C（変化率） |
| 電流 | A（絶対値）＋ D（移動平均） |
| 温度 | A（絶対値・安全上限）＋ B（季節別） ＋ D（移動平均） |
| 振動 | C（変化率）＋ G（単一ML） |
| 流量 | C（変化率）＋ G（単一ML） |

### 二次診断

| 目的 | 推奨パターン |
|------|-------------|
| 全センサーの相関異常・予兆検知 | I（多変量ML） |
| 重篤度の自動分類 | J（異常スコア分類） |
| 故障種別の絞り込み | E（複合条件判定） |
