# ELK スタック 振動異常検知 設定まとめ

## 概要

Elastic ML の異常検知ジョブ `vibration-anomaly-detection` を使い、振動データの上限・下限境界を監視する SIEM 検出ルールを設定する。

---

## 環境・制約

| 項目 | 内容 |
|---|---|
| ライセンス | Basic |
| ジョブ名 | vibration-anomaly-detection |
| bucket_span | 900s（15分） |
| model_plot_config | Basic ライセンスのため使用不可 |
| 境界値の保存先 | `.ml-anomalies-shared` の `anomaly_score_explanation` |

---

## 境界値フィールドの場所

Basic ライセンスでは `.ml-model-plot-*` は作成されない。代わりに `.ml-anomalies-shared` の `record` ドキュメント内の `anomaly_score_explanation` に境界値が格納されている。

| フィールド | 内容 |
|---|---|
| `anomaly_score_explanation.upper_confidence_bound` | 上の境界 |
| `anomaly_score_explanation.lower_confidence_bound` | 下の境界 |
| `anomaly_score_explanation.typical_value` | 正常中央値 |
| `actual` | 実測値 |

### 確認コマンド

```json
GET .ml-anomalies-shared/_search
{
  "size": 1,
  "query": {
    "bool": {
      "filter": [
        { "term": { "job_id": "vibration-anomaly-detection" } },
        { "term": { "result_type": "record" } }
      ]
    }
  },
  "_source": [
    "timestamp",
    "actual",
    "anomaly_score_explanation"
  ]
}
```

## SIEM 検出ルール設定

### ルール概要

| 項目 | 内容 |
|---|---|
| ルールタイプ | Custom query |
| 対象インデックス | `.ml-anomalies-shared` |
| 検知条件 | 下の境界（lower_confidence_bound）が 3 を超過 |

### Kibana UI 手順

```
Security > Alerts > Manage rules > Create new rule
→ Rule type: Custom query
→ Index patterns: .ml-anomalies-shared
→ Custom query (KQL):
    job_id : "vibration-anomaly-detection" and result_type : "record"
→ Filters > Add filter > Edit as Query DSL
    （下記 Query DSL を貼り付け）
→ About rule: 名前・severity・risk_score を入力
→ Schedule: Runs every 15m / Additional look-back time 1m
→ Create & enable rule
```

### Filter（Query DSL）

```json
{
  "range": {
    "anomaly_score_explanation.lower_confidence_bound": {
      "gt": 3
    }
  }
}
```

### About rule 設定値

| 項目 | 値 |
|---|---|
| Name | vibration: 下の境界が3を超過 |
| Description | lower_confidence_bound が3を超えた場合に検知 |
| Severity | High |
| Risk score | 65 |
| Tags | vibration, lower-bound, threshold-3 |

### Schedule 設定値

| 項目 | 値 |
|---|---|
| Runs every | 15m |
| Additional look-back time | 1m |

## 参考：その他の検知パターン

### 上の境界を actual が超えたら検知

**Filter（Query DSL）**

```json
{
  "script": {
    "script": {
      "source": "doc['actual'].size() > 0 && doc['anomaly_score_explanation.upper_confidence_bound'].size() > 0 && doc['actual'][0] > doc['anomaly_score_explanation.upper_confidence_bound'].value",
      "lang": "painless"
    }
  }
}
```

### actual が typical より3以上上昇したら検知

**Filter（Query DSL）**

```json
{
  "script": {
    "script": {
      "source": "doc['actual'].size() > 0 && doc['typical'].size() > 0 && (doc['actual'][0] - doc['typical'][0]) > 3",
      "lang": "painless"
    }
  }
}
```

### 上下どちらの境界超過も検知

**Filter（Query DSL）**

```json
{
  "bool": {
    "filter": [
      { "term": { "job_id": "vibration-anomaly-detection" } },
      { "term": { "result_type": "record" } }
    ],
    "should": [
      {
        "script": {
          "script": {
            "source": "doc['actual'].size() > 0 && doc['anomaly_score_explanation.upper_confidence_bound'].size() > 0 && doc['actual'][0] > doc['anomaly_score_explanation.upper_confidence_bound'].value",
            "lang": "painless"
          }
        }
      },
      {
        "script": {
          "script": {
            "source": "doc['actual'].size() > 0 && doc['anomaly_score_explanation.lower_confidence_bound'].size() > 0 && doc['actual'][0] < doc['anomaly_score_explanation.lower_confidence_bound'].value",
            "lang": "painless"
          }
        }
      }
    ],
    "minimum_should_match": 1
  }
}
```

## トラブルシューティング

### model_plot インデックスが作成されない

| 原因 | 対処 |
|---|---|
| Basic ライセンス | model_plot は Platinum 以上が必要。`anomaly_score_explanation` を代替として使用する |
| ジョブ停止中に update していない | datafeed 停止 → ジョブ close → update → open → datafeed start の順で実施 |

### SIEM ルールが検知しない

| 原因 | 対処 |
|---|---|
| from の時間範囲が狭い | `from: now-7d` など広げる |
| データフィードにデータが来ていない | センサー・Beats 側の疎通確認 |
| インデックス名が違う | `GET _cat/indices/.ml-*` で実際の名前を確認 |

### データフィード・ジョブの再起動手順

```json
POST _ml/datafeeds/datafeed-vibration-anomaly-detection/_stop
POST _ml/anomaly_detectors/vibration-anomaly-detection/_close
{
  "force": true
}
POST _ml/anomaly_detectors/vibration-anomaly-detection/_update
{
  "model_plot_config": {
    "enabled": true,
    "annotations_enabled": true
  }
}
POST _ml/anomaly_detectors/vibration-anomaly-detection/_open
POST _ml/datafeeds/datafeed-vibration-anomaly-detection/_start
```
