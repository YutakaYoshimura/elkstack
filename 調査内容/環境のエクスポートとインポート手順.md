# Podman コンテナ全体エクスポート・復旧手順

## 概要

`podman save` はイメージ（設計図）のみを保存します。インデックスやダッシュボードなどの**データはボリュームに保存**されているため、完全な環境移行にはイメージ・ボリューム・コンテナ設定の3点セットが必要です。

| ファイル | 内容 |
|------|------|
| `all_images.tar` | コンテナイメージ一式 |
| `vol_<名前>.tar` | ボリュームデータ（インデックス・ダッシュボード等） |
| `kube_<名前>.yaml` | コンテナ起動設定 |

---

## エクスポート手順（移行元マシン）

### 1. コンテナを停止する

データ破損を防ぐため、必ず停止してからエクスポートします。

```bash
podman stop $(podman ps -q)
```

### 2. 全イメージをエクスポートする

```bash
podman save $(podman images --format "{{.Repository}}:{{.Tag}}" \
  | grep -v "<none>:<none>") \
  -o all_images.tar
```

> ⚠️ `podman save $(podman images -q)` はタグ情報が失われるため非推奨です。

### 3. 全ボリュームをエクスポートする

```bash
for vol in $(podman volume ls -q); do
  podman volume export "$vol" -o "vol_${vol}.tar"
done
```

### 4. コンテナ設定をエクスポートする

```bash
for ctr in $(podman ps -a --format "{{.Names}}"); do
  podman generate kube "$ctr" > "kube_${ctr}.yaml"
done
```

### 5. 納品ファイルをまとめる

```bash
mkdir -p podman_export
mv all_images.tar podman_export/
mv vol_*.tar podman_export/
mv kube_*.yaml podman_export/
```

**納品物の確認**

```bash
ls podman_export/
# all_images.tar
# vol_elasticsearch-data.tar  （例）
# kube_elasticsearch.yaml     （例）
# kube_kibana.yaml            （例）
# kube_logstash.yaml          （例）
```

## 復旧手順（移行先マシン）

### 1. Podmanのインストール確認

```bash
podman --version
```

### 2. イメージをロードする

```bash
podman load -i podman_export/all_images.tar
```

### 3. ボリュームを復元する

```bash
for f in podman_export/vol_*.tar; do
  vol_name="${f#podman_export/vol_}"; vol_name="${vol_name%.tar}"
  podman volume create "$vol_name"
  podman volume import "$vol_name" "$f"
done
```

### 4. コンテナを起動する

```bash
for yaml in podman_export/kube_*.yaml; do
  podman kube play "$yaml"
done
```

### 5. 動作確認

```bash
# コンテナの起動状態を確認
podman ps

# ボリュームの確認
podman volume ls
```

---

## 注意事項

| 項目 | 内容 |
|------|------|
| コンテナの停止 | エクスポート前に必ず `podman stop` を実行する |
| バージョン一致 | 移行元と移行先でPodman・イメージのバージョンを揃える |
| `podman generate kube` | Podman v4以降が必要 |
| コンテナの実行状態 | メモリ上の状態は保存されない（ファイル・DBデータはボリュームで保持） |
| ELKスタックの場合 | Elasticsearchのバージョンが異なるとボリュームが読み込めない場合がある |

## 一括スクリプト（参考）

### エクスポートスクリプト

```bash
#!/bin/bash
# export_all.sh

set -e
EXPORT_DIR="./podman_export"
mkdir -p "$EXPORT_DIR"

echo "=== コンテナを停止 ==="
podman stop $(podman ps -q) 2>/dev/null || true

echo "=== イメージをエクスポート ==="
podman save $(podman images --format "{{.Repository}}:{{.Tag}}" \
  | grep -v "<none>:<none>") \
  -o "$EXPORT_DIR/all_images.tar"

echo "=== ボリュームをエクスポート ==="
for vol in $(podman volume ls -q); do
  podman volume export "$vol" -o "$EXPORT_DIR/vol_${vol}.tar"
  echo "  exported: $vol"
done

echo "=== コンテナ設定をエクスポート ==="
for ctr in $(podman ps -a --format "{{.Names}}"); do
  podman generate kube "$ctr" > "$EXPORT_DIR/kube_${ctr}.yaml"
  echo "  exported: $ctr"
done

echo "=== 完了: $EXPORT_DIR ==="
ls "$EXPORT_DIR"
```

### 復旧スクリプト

```bash
#!/bin/bash
# restore_all.sh

set -e
EXPORT_DIR="./podman_export"

echo "=== イメージをロード ==="
podman load -i "$EXPORT_DIR/all_images.tar"

echo "=== ボリュームを復元 ==="
for f in "$EXPORT_DIR"/vol_*.tar; do
  vol_name="${f#$EXPORT_DIR/vol_}"; vol_name="${vol_name%.tar}"
  podman volume create "$vol_name" 2>/dev/null || true
  podman volume import "$vol_name" "$f"
  echo "  restored: $vol_name"
done

echo "=== コンテナを起動 ==="
for yaml in "$EXPORT_DIR"/kube_*.yaml; do
  podman kube play "$yaml"
  echo "  started: $yaml"
done

echo "=== 完了 ==="
podman ps
```
