# ElastAlert 設定手順まとめ

## 1. GmailでSMTP送信する設定

### Gmail側の準備（アプリパスワードの生成）
1. Googleアカウント → セキュリティ → 2段階認証プロセスを有効化
2. セキュリティ → アプリパスワード → 新しいアプリパスワードを生成
3. 生成された16文字のパスワードをメモしておく

### config.yaml（グローバル設定）

```yaml
es_host: localhost
es_port: 9200

rules_folder: rules
run_every:
  minutes: 1

buffer_time:
  minutes: 15

smtp_host: smtp.gmail.com
smtp_port: 587
smtp_ssl: false
smtp_auth_file: /opt/elastalert/smtp_auth.yaml
from_addr: your-email@gmail.com
timezone: "Asia/Tokyo"
```

### smtp_auth.yaml（認証情報）

```yaml
user: "your-email@gmail.com"
password: "xxxx xxxx xxxx xxxx"  # アプリパスワード（16文字）
```

パーミッションを制限する：

```bash
chmod 600 /opt/elastalert/smtp_auth.yaml
```

## 2. 過去データでテストする方法

### elastalert-test-ruleコマンド

```bash
# 検知確認のみ（メール送信なし）
elastalert-test-rule rules/my_rule.yaml \
  --config config.yaml \
  --start "2024-01-01T00:00:00" \
  --end "2024-01-02T00:00:00"

# 実際にメール送信
elastalert-test-rule rules/my_rule.yaml \
  --config config.yaml \
  --start "2024-01-01T00:00:00" \
  --end "2024-01-02T00:00:00" \
  --alert
```

### 目的別コマンド早見表

| 目的 | コマンド/設定 |
|------|------------|
| 検知だけ確認 | `elastalert-test-rule`（`--alert`なし） |
| 実際にメール送信 | `elastalert-test-rule --alert` |
| メール内容だけ確認 | `alert: debug` に変更 |
| 本番同様に過去から流す | `elastalert --start "..."` |

## 3. MLジョブの異常スコア監視ルール（job_a）

```yaml
name: job_a anomaly score alert
type: any

es_host: localhost
es_port: 9200

index: .ml-anomalies-*

run_every:
  minutes: 5

buffer_time:
  minutes: 15

filter:
  - term:
      job_id: "job_a"
  - term:
      result_type: "record"
  - range:
      record_score:
        gte: 75

alert:
  - email

email:
  - recipient@example.com

from_addr: your-email@gmail.com
smtp_host: smtp.gmail.com
smtp_port: 587
smtp_auth_file: /opt/elastalert/smtp_auth.yaml

alert_subject: "[異常検知] job_a 異常スコア {0} を検知"
alert_subject_args:
  - record_score

alert_text: |
  job_aで高異常スコアを検知しました。

  ■ 検知情報
  - ジョブID       : {0}
  - 異常スコア     : {1}
  - 検知時刻       : {2}
  - 対象フィールド : {3}
  - 実際の値       : {4}
  - 典型値（平均） : {5}

alert_text_args:
  - job_id
  - record_score
  - timestamp
  - function
  - actual
  - typical

alert_text_type: alert_text_only

realert:
  minutes: 60
```

## 4. 元ドキュメントの値をメールに表示する

### Jinja2テンプレートを使う方法（推奨）

```yaml
alert_text_type: alert_text_jinja2

alert_text: |
  ■ 検知情報
  - デバイスID : {{ device_id }}
  - 振動値     : {{ vibration }}
  - 温度       : {{ temperature }}
  - 検知時刻   : {{ timestamp_jst }}

  {% if vibration >= 1500 %}
  ⚠️ 緊急: 即時対応が必要です
  {% elif vibration >= 1200 %}
  ⚡ 重大: 早急に確認してください
  {% else %}
  ℹ️ 警告: 監視を強化してください
  {% endif %}
```

`alert_text_args`への個別指定が不要で、`{{ フィールド名 }}`で直接アクセスできる。

| 比較 | `alert_text_args` | `Jinja2` |
|------|------------------|----------|
| フィールド指定 | リストに順番通り列挙が必要 | `{{ フィールド名 }}`で直接指定 |
| 存在しないフィールド | エラーになる | `\| default('なし')`で安全に処理 |
| 条件分岐 | 不可 | `if`文が使える |
| 記述量 | 多い | 少ない |

## 5. 振動値の固定閾値監視ルール

### anyタイプ（最もシンプル）

```yaml
name: vibration threshold alert
type: any

index: sensor-*

filter:
  - term:
      device_id.keyword: "pump-01"
  - range:
      vibration:
        gte: 1000

alert:
  - email

email:
  - recipient@example.com

from_addr: your-email@gmail.com
smtp_host: smtp.gmail.com
smtp_port: 587
smtp_auth_file: /opt/elastalert/smtp_auth.yaml

alert_subject: "【振動異常アラート】{{ device_id }} で振動上昇を検知"

alert_text_type: alert_text_jinja2

alert_text: |
  振動値が閾値（1000）を超えました。

  ■ 検知情報
  - デバイスID : {{ device_id }}
  - 振動値     : {{ vibration }}
  - 温度       : {{ temperature | default('N/A') }}
  - 設置場所   : {{ location | default('N/A') }}
  - 検知時刻   : {{ timestamp_jst | default(timestamp) }}

realert:
  minutes: 30
```

### タイプ選択の目安

| 状況 | 推奨タイプ |
|------|-----------|
| 1回でも超えたら即通知 | `any` |
| 平均値で判断したい | `metric_aggregation` |
| 複数回超えたら通知（誤検知対策） | `frequency` |
| 急激な変化を検知したい | `spike` |

### 複数デバイスに絞る場合

```yaml
filter:
  - terms:
      device_id.keyword:
        - "pump-01"
        - "pump-02"
        - "pump-03"
  - range:
      vibration:
        gte: 1000
```

### デバイスごとに閾値を変える場合

```bash
rules/
├── vibration_pump-01.yaml   # pump-01: 閾値1000
├── vibration_pump-02.yaml   # pump-02: 閾値1500
└── vibration_pump-03.yaml   # pump-03: 閾値800
```

## 7. 対象データの事前確認

### ヒット件数の確認

```bash
curl -X GET "localhost:9200/sensor-*/_count?pretty" \
  -H "Content-Type: application/json" \
  -d '{
    "query": {
      "bool": {
        "filter": [
          { "term":  { "device_id.keyword": "pump-01" }},
          { "range": { "vibration": { "gte": 1000 }}}
        ]
      }
    }
  }'
```

### 確認の流れ

```
① _countでヒット件数を確認
        ↓
件数 > 0  →  ② _searchで実際のデータ内容を確認
        ↓
データOK  →  ③ elastalert-test-ruleで検知テスト（メール送信なし）
        ↓
検知OK   →  ④ --alertオプションつきで実際にメール送信テスト
```

## 8. smtp_auth.yamlが存在しない場合の対処（Podman/Dockerコンテナ環境）

### PowerShellでファイルを作成してコンテナにコピーする手順

**① PowerShellでsmtp_auth.yamlを作成**

```powershell
@"
user: "your-email@gmail.com"
password: "xxxx xxxx xxxx xxxx"
"@ | Out-File -FilePath "$env:USERPROFILE\smtp_auth.yaml" -Encoding utf8
```

**② コンテナにコピー**

```powershell
podman cp $env:USERPROFILE\smtp_auth.yaml elastalert:/opt/elastalert/smtp_auth.yaml
```

**③ コンテナ内で確認**

```powershell
podman exec -it elastalert cat /opt/elastalert/smtp_auth.yaml
```

**④ ルールファイルのsmtp_auth_fileのパスを修正**

```powershell
podman exec -it elastalert sed -i 's|/etc/elastalert/smtp_auth.yaml|/opt/elastalert/smtp_auth.yaml|g' /opt/elastalert/rules/anomaly-vibration--threshold-pump-01.yaml
```

**⑤ テスト実行**

```powershell
podman exec -it elastalert elastalert-test-rule /opt/elastalert/rules/anomaly-vibration--threshold-pump-01.yaml --config /opt/elastalert/config.yaml --alert --start "2026-03-18T00:00:00" --end "2026-03-19T00:00:00"
```

## 9. よくあるトラブルと対処法

| エラーメッセージ | 原因 | 対処 |
|----------------|------|------|
| `SMTPAuthenticationError` | アプリパスワードが間違い | アプリパスワードを再確認 |
| `Connection refused` | SMTPホスト・ポート設定ミス | `smtp_host`, `smtp_port`を確認 |
| `No such file or directory`（smtp_auth.yaml） | ファイルが存在しない | ファイルを作成してパスを修正 |
| `Permission denied` | 書き込み権限がない | コンテナ外からコピーする |
| `0 matches` | 対象データが見つからない | インデックス名・フィールド名・期間を確認 |
| realertが効いている | 前回アラートから待機中 | `elastalert_status`インデックスをリセット |

### realertのリセット

```bash
curl -X DELETE "localhost:9200/elastalert_status"
```
